================================================================================
                            DATA FLOW: BACKUP
================================================================================

    User creates client/loan in app
              │
              ▼
    Data stored in Browser localStorage
              │
              ▼
    Every 5 minutes (or manual click)
              │
              ▼
    supabase.js triggers backupData()
              │
              ▼
    Gathers clients, loans, payments, settings
              │
              ▼
    POST to /api/sync/backup with:
    • Authorization: Bearer JWT_TOKEN
    • X-User-Email: user@example.com
    • Body: {clients: [...], loans: [...], ...}
              │
              ▼
    Backend receives request
              │
              ▼
    supabase_service.save_backup()
              │
              ▼
    Upserts to app_backups table in PostgreSQL
              │
              ▼
    Data persisted in Supabase cloud
              │
              ▼
    ✅ Notification: "Data synced with cloud"


================================================================================
                           DATA FLOW: RESTORE
================================================================================

    User clicks "Restore from Cloud"
              │
              ▼
    Frontend calls supabaseIntegration.manualRestore()
              │
              ▼
    GET /api/sync/restore with:
    • Authorization: Bearer JWT_TOKEN
    • X-User-Email: user@example.com
              │
              ▼
    Backend queries supabase_service.get_backup()
              │
              ▼
    Fetches from app_backups table
              │
              ▼
    Returns: {clients: [...], loans: [...], ...}
              │
              ▼
    Frontend updates localStorage with cloud data
              │
              ▼
    Page reloads to display restored data
              │
              ▼
    ✅ Data restored from cloud

                        PERFORMANCE METRICS
================================================================================

Backup Size:
  Typical: 50KB - 500KB per user
  Maximum: 1GB (Supabase limit)
  Compression: JSONB format reduces size

Sync Speed:
  Network upload: ~1-5 seconds
  Database write: ~100-500ms
  Total time: ~2-5 seconds per backup

Frequency:
  Automatic: Every 5 minutes
  Manual: On demand
  Network: Only if data changed (optimization ready)

Storage:
  Per user: ~1MB average
  1000 users: ~1GB
  5 years: ~200GB (with retention policy)
