API ENDPOINTS
═══════════════════════════════════════════════════════════════════

Authentication: Bearer token required in Authorization header

POST /api/whatsapp/setup
  - Setup WhatsApp instance
  - Params: { phone: "+91XXXXXXXXXX" }
  - Returns: { success, instance_name }

GET /api/whatsapp/qr
  - Get QR code for scanning
  - Returns: { success, qr: "base64_data" }

GET /api/whatsapp/status
  - Check connection status
  - Returns: { success, connected: boolean }

POST /api/whatsapp/disconnect
  - Logout and disconnect
  - Returns: { success, message }

POST /api/whatsapp/test
  - Send test message
  - Params: { phone: "+91XXXXXXXXXX" }
  - Returns: { success, message }

GET /api/whatsapp/reminders/config
  - Get reminder configuration
  - Returns: { success, config: {...} }

POST /api/whatsapp/reminders/config
  - Update reminder settings
  - Params: { 
      due_today: boolean,
      due_tomorrow: boolean,
      overdue: boolean,
      phone: "+91XXXXXXXXXX"
    }
  - Returns: { success, message }

HOW IT WORKS
═══════════════════════════════════════════════════════════════════

USER CONNECTS WHATSAPP:
  1. User opens app → Settings → WhatsApp Automation
  2. Enters phone number + clicks "Connect WhatsApp"
  3. Frontend calls /api/whatsapp/setup endpoint
  4. Backend creates Evolution API instance
  5. QR code displayed on screen
  6. User scans with phone camera/WhatsApp
  7. Backend confirms connection status
  8. UI updates to "Connected" state

DAILY REMINDERS RUN (9 AM):
  1. Scheduler triggers daily job
  2. Gets all connected users
  3. Fetches their active loans
  4. Calculates next due date per loan
  5. Sends reminders based on settings:
     - Due today: sends payment reminder
     - Due tomorrow: sends upcoming payment alert
     - Overdue: sends urgent reminder
  6. Logs all sent messages

USER SENDS MANUAL REMINDER:
  1. User clicks WhatsApp button on loan
  2. Frontend gets client phone number
  3. Calls /api/whatsapp/test endpoint
  4. Backend sends message via Evolution API
  5. Returns success/failure status
  6. User sees confirmation toast


TROUBLESHOOTING QUICK GUIDE
═══════════════════════════════════════════════════════════════════

Check Backend Logs:
  • Look for "WhatsApp reminder scheduler started"
  • Check for any API connection errors
  • Verify Evolution API URL is responding

Check Frontend Console:
  • Open DevTools (F12) → Console tab
  • Look for WhatsApp Automation initialization
  • Check for API response errors

Test API Connectivity:
  • Run: python test_whatsapp_api.py
  • Should show API is accessible
  • Verify credentials are loaded

Database Verification:
  • In Supabase, query: SELECT * FROM kf_whatsapp_config;
  • Should show connected users

Common Issues:
  • "API credentials not configured" → Check .env file
  • "Connection timeout" → Verify API server URL
  • "QR not displaying" → Check browser console for errors
  • "Message not sending" → Verify phone format (+91XXXXXXXXXX)
